§ 01

mxcli 是什么？解决什么问题？

如果你用过 Mendix Studio Pro，你一定知道这个现实：Mendix 项目本质上是一个二进制 .mpr 文件。这个文件对人类友好（Studio Pro 帮你可视化了一切），但对 AI 助手和自动化工具来说，就是一个不透明的黑盒。

mxcli 就是来打破这堵墙的。它是由 Mendix Labs 开源的命令行工具，核心目标是：让 Claude Code、Cursor、Copilot 等 AI 编程助手能够真正读懂并操作 Mendix 项目。

🔍
MDL 查询语言
类 SQL 语法，以文本方式描述和查询 Mendix 模型元素
🤖
AI 工具集成
自动生成 CLAUDE.md、AGENTS.md，让 AI 理解项目上下文
🛡️
Dev Container
沙盒环境，限制 AI Agent 的文件访问范围，防止意外改动
✅
Starlark Linter
自定义代码质量规则，生成 HTML 报告，支持 CI/CD 集成
🧩
VSCode 扩展
项目结构可视化，无需打开 Studio Pro 即可浏览文档树
🚀
PAD 本地运行
Docker 容器内启动 Mendix Runtime + PostgreSQL，直接测试
⚠️ Alpha 警告：mxcli 目前处于早期阶段，由官方团队用 AI 编程助手"高速开发"完成。可能存在 bug，且有损坏项目文件的风险。当前仅在 Mendix 11.6 上经过验证。请务必在副本或 Git 分支上操作，绝不直接操作生产项目。
§ 02

核心架构：MDL + Agent 集成

理解 mxcli 的核心，要从它的架构思路出发。

MDL（Mendix Definition Language） 是整个工具的技术核心。它提供了一套类 SQL 的语法，让你可以像查数据库一样查询 Mendix 项目的内部结构 —— 查看 Entity、Microflow、Page，并对其进行修改。

MDL 示例语句
-- 列出所有模块 SHOW MODULES -- 查看指定模块下所有 Entity DESCRIBE MyModule.Customer -- 获取页面的 MDL 源码 SOURCE MyModule/CustomerOverview -- 列出模块中所有 Microflow LIST microflows FROM MyModule

§ 04

安装 mxcli

mxcli 以预编译二进制文件形式分发，无需安装 Go 运行时，直接下载对应平台的可执行文件即可。

PowerShell
# 1. 从 GitHub Releases 下载最新 Windows 二进制 Invoke-WebRequest -Uri "https://github.com/mendixlabs/mxcli/releases/latest/download/mxcli-windows-amd64.exe" -OutFile "mxcli.exe" # 2. 将可执行文件移动到系统 PATH 目录（推荐） Move-Item mxcli.exe C:\Windows\System32\mxcli.exe # 或者放在自定义目录并添加到 PATH $env:PATH += ";" # 3. 验证安装 mxcli --version # 预期输出: mxcli version 0.x.x
zsh / bash
# 1. 下载 macOS 版本（Apple Silicon 或 Intel） # Apple Silicon (M1/M2/M3): curl -L https://github.com/mendixlabs/mxcli/releases/latest/download/mxcli-darwin-arm64 -o mxcli # Intel Mac: curl -L https://github.com/mendixlabs/mxcli/releases/latest/download/mxcli-darwin-amd64 -o mxcli # 2. 添加可执行权限 chmod +x mxcli # 3. 移动到 PATH（以便全局使用） sudo mv mxcli /usr/local/bin/mxcli # 4. macOS 可能提示"无法验证开发者"，执行以下命令解除 xattr -d com.apple.quarantine /usr/local/bin/mxcli # 5. 验证安装 mxcli --version
bash
# 1. 下载 Linux amd64 二进制 curl -L https://github.com/mendixlabs/mxcli/releases/latest/download/mxcli-linux-amd64 -o mxcli # 2. 添加可执行权限 chmod +x mxcli # 3. 移动到系统 PATH sudo mv mxcli /usr/local/bin/mxcli # 4. 验证安装 mxcli --version # mxcli version 0.x.x
📦 Releases 页面：所有平台的最新二进制文件都可以在 github.com/mendixlabs/mxcli/releases 找到。如果上方 URL 与实际不符，请前往该页面手动下载对应平台的文件。
§ 05

初始化你的第一个 Mendix 项目

安装完成后，第一步是对已有的 Mendix 项目运行 mxcli init。这个命令会自动生成所有 AI 工具所需的配置文件。

准备工作：复制项目副本

bash
# 永远在副本或 Git 分支上操作！ cp -r ~/Mendix/MyApp ~/Mendix/MyApp-mxcli-test cd ~/Mendix/MyApp-mxcli-test # 或者用 Git（推荐） cd ~/Mendix/MyApp git checkout -b feature/mxcli-experiment

运行初始化命令

bash
# 为 Claude Code 初始化（同时支持 --tool cursor, --tool copilot 等） mxcli init --tool claude "C:\Users\YourName\Mendix\MyApp" # Linux / macOS 路径格式 mxcli init --tool claude "/home/user/Mendix/MyApp" # 执行成功后，你会看到类似输出： ✓ Created 41 skill files in .ai-context/skills/ ✓ Created .claude/settings.json ✓ Created CLAUDE.md ✓ Created 10 command files in .claude/commands/ ✓ Created 29 lint rule files in .claude/lint-rules/ ✓ Created AGENTS.md ✓ Created .devcontainer/ configuration ✓ Initialization complete!

初始化生成的文件结构含义如下：

项目目录结构
MyApp/ ├── CLAUDE.md # 告诉 Claude Code 如何与此项目交互 ├── AGENTS.md # 通用 AI Agent 上下文文档 ├── .ai-context/ │ └── skills/ # 41 个技能文件：教 AI 如何构建 Mendix 组件 ├── .claude/ │ ├── settings.json # Claude Code 设置 │ ├── commands/ # 10 个自定义命令 │ └── lint-rules/ # 29 条 Mendix 质量规则 ├── .devcontainer/ │ └── devcontainer.json # Dev Container 沙箱配置 └── MyApp.mpr # 原始 Mendix 项目文件
§ 06

用 MDL 语言查询项目结构

mxcli 提供了两种使用 MDL 的方式：单行命令模式 和 REPL 交互模式（类似 psql 或 sqlplus）。

📌 单行命令模式

bash
# 基本语法：mxcli -p <项目路径> -c "" # 列出所有模块 mxcli -p MyApp.mpr -c "SHOW MODULES" Administration MyFirstModule System # 查看某模块的文档树结构 mxcli -p MyApp.mpr -c "STRUCTURE MyFirstModule" # 查看某个 Entity 的详细定义 mxcli -p MyApp.mpr -c "DESCRIBE MyFirstModule.Customer" # 获取某个 Page 的 MDL 源码 mxcli -p MyApp.mpr -c "SOURCE MyFirstModule/CustomerOverview_NewEdit" # 列出某模块下所有 Microflow mxcli -p MyApp.mpr -c "LIST microflows FROM MyFirstModule"

💬 REPL 交互模式

REPL 模式适合连续探索项目，不用每次都指定 -p 参数。

bash
# 进入 REPL 模式 mxcli -p MyApp.mpr mxcli> SHOW MODULES Administration MyFirstModule System mxcli> LIST pages FROM MyFirstModule CustomerOverview CustomerDetail Home_Web mxcli> DESCRIBE MyFirstModule.Order Entity: Order Attributes: - OrderId : AutoNumber - Status : Enum (New, Processing, Shipped) - TotalAmount : Decimal - CreatedDate : DateTime Associations: - Order_Customer → Customer (ManyToOne) mxcli> .exit
§ 07

运行 Lint 和质量报告

mxcli 内置了一套基于 Starlark 的 linter，可以检查常见的 Mendix 模型问题，并生成 HTML 格式的质量报告。

bash
# 运行 lint 检查（输出到终端） mxcli lint -p MyApp.mpr Checking module: MyFirstModule ⚠ MF001: Microflow 'ACT_Customer_Create' has no error handling ⚠ PG002: Page 'CustomerOverview' has more than 20 data grid columns ✓ ENT003: All entities have documentation Lint complete: 2 warnings, 0 errors # 生成 HTML 报告（方便分享给团队） mxcli report -p MyApp.mpr --format html --output report.html ✓ Report generated: report.html # 只检查特定模块 mxcli lint -p MyApp.mpr --module MyFirstModule
✅ CI/CD 集成提示：将 mxcli lint -p yourapp.mpr 加入你的 CI 流水线，可以在每次合并前自动检查 Mendix 模型质量。返回非零 exit code 即代表有错误。
§ 08

与 Claude Code 协同开发

这是 mxcli 最令人兴奋的部分。mxcli init 生成的 CLAUDE.md 文件，让 Claude Code 在启动时就能理解整个项目的上下文。

bash
# 1. 在项目目录启动 Claude Code cd ~/Mendix/MyApp claude # Claude Code 会自动读取 CLAUDE.md 并告知其可用的 MDL 命令 # 此后你可以直接用自然语言指令，例如： You: 请帮我查看 MyFirstModule 下有哪些 Entity，然后为 Customer 实体生成一个列表页面 Claude: 我来先用 mxcli 探索项目结构... $ mxcli -p MyApp.mpr -c "LIST entities FROM MyFirstModule" 找到以下实体：Customer, Order, Product, Category 接下来我将生成 Customer 的列表页面...
🧠 工作原理：Claude Code 读取 CLAUDE.md 后，知道可以调用 mxcli 命令来搜索项目结构、获取 MDL 源码，然后结合 .ai-context/skills/ 中的技能文件，生成符合 Mendix 最佳实践的代码改动。AI 不是在"猜"你的项目结构，而是真正在"读"它。

一个完整的 AI 协作场景示例（由社区实测完成）：在几小时内，Claude Code 借助 mxcli 构建了一个包含 12 个实体、25 个 Microflow、17 个管理页面的完整业务系统 —— 全程无需打开 Studio Pro。

§ 09

进阶：Dev Container 沙箱环境

如果你计划长期让 AI Agent 自主修改项目，强烈建议使用 Dev Container 模式。它通过 Docker 隔离 AI 的操作权限，防止意外修改项目之外的系统文件。

bash
# 前置：确保 Docker Desktop 已运行 # 1. 用 VS Code 打开项目（已有 .devcontainer/ 配置） code ~/Mendix/MyApp # 2. VS Code 检测到 .devcontainer/ 后会弹窗提示 # 点击 "Reopen in Container" 即可进入沙箱 # 3. 容器启动后，mxcli 已预装在容器内 # 在容器内的终端中可直接使用 root@container:~# mxcli --version mxcli version 0.x.x # 4. （可选）通过 PAD 启动 Mendix Runtime 进行测试 mxcli start -p MyApp.mpr ✓ Starting Mendix Runtime on port 8080... ✓ PostgreSQL ready on port 5432 ✓ Application available at http://localhost:8080
🔒 沙箱的意义：Dev Container 会将 AI Agent 的文件系统访问限制在挂载的项目目录内。这意味着即使 AI 产生了"出格"的文件操作指令，也不会影响你的主机系统，只有 /workspaces/MyApp 目录内的文件会被修改。
§ 10

当前限制与避坑指南

作为 Alpha 工具，mxcli 目前有一些明确的边界需要了解：

功能 状态 建议
读取 / 探索项目结构 ✅ 稳定可用 放心使用，这是最成熟的功能
Lint 质量报告 ✅ 稳定可用 可集成到 CI/CD
MDL 作为查询语言 ✅ 核心创新 语法仍在迭代，关注 Changelog
AI Agent 集成概念 ⚠️ 实验阶段 适合尝鲜，不建议用于生产开发
写入 / 修改项目文件 ⚠️ 谨慎使用 必须在 Git 分支或副本上操作
Mendix 版本兼容性 ⚠️ 仅 11.6 验证 其他版本可试，但可能有问题
Dev Container + PAD ⚠️ 需要 Docker Windows 用户需先配置好 WSL 2
🛑 黄金规则：在使用 mxcli 的任何写入操作之前，先提交你的 Git 变更。如果项目文件损坏，git checkout -- . 是你的最后防线。无论如何，永远不要直接在生产项目上运行 mxcli。
